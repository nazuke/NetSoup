What is NetSoup Anyway?

NetSoup is heavily under development.

NetSoup is an organised collection of my own Perl class libraries. Most of the code in there is geared towards manipulating data sourced from, or destined for, the Internet. Most notably, HTML.

For instance, there's a, mostly working, XML parser and DOM2 compiler; a few bits of HTTP client code, and some CGI tools amongst other things. The direction of development has mostly been dictated by my working requirements.

However, one consistent project I've been striving towards has been HyperGlot.

HyperGlot is aiming to be a localisation system for XML/XHTML based web sites. The XML parser has enough functionality to be usable, and there's a simple translation memory client/server system in place. I'm just now starting work on the user interface elements, which will be written in Java.

For more information, download the archive and check out the code yourself.

System Requirements

NetSoup was developed under RedHat Linux 5.1 with Perl version 5.005_650. I always try to keep up to date with the latest Perl version. My particular Perl was built with threads enabled, so at some point I may write some classes that require a threaded Perl. Using an earlier version of Perl should not cause too many problems, but don't go any lower than 5.004_xx.

Although developed under a Unix-style system, many of the classes are not necessarily Unix dependent. However, if you do intend to use these classes under some other system then please rummage through the code that deals with files, directories and pathnames; these pretty much contain stuff that is not very portable. I have made some attempt to detect the host system, and deal with it, but this is not a priority for me. As time goes on, NetSoup will almost definitely become more Unix dependant.

To use the NetSoup classes, you basically need Perl installed, preferably under a Unix style system, and of course the ability to code in Perl.

Installation

I'm afraid that the installation procedure only works on a Unix-style system, even then it has only been tested on Linux.

Please read the INSTALL file at the top level of the archive for installation instructions.

The installation procedure is evolving rapidly, so make sure you read the instructions carefully before proceeding. And never install untested versions on a production installation.
